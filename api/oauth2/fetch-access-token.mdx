## Description

After accessing the ZEBEDEE login portal URL and successfully authenticating, the user will be redirected back to your application with `state` and `code` query parameters included. These two properties allow you to hit the `token` endpoint, and get the user's access token.

Once you have the `code` returned from the previous `authorization` endpoint, you must now make a `POST` request to get the token from this endpoint.

## Usage

[Read here for more details](/docs/zbd-login/intro) on how to setup ZBD Login (OAuth2) with your platform.
